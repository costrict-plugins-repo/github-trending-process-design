# /scale-program
Turn something that worked into something repeatable.
## Steps
1. **Diagnose first** — Confirm the current process is worth scaling using `process-retrospective` skill.
2. **Codify** — Write the reusable playbook using `scaling-playbook` skill.
3. **Set guardrails** — Define governance for many teams using `governance-model` skill.
## Output
A scaling pack: a process diagnosis, a reusable playbook with templates, and lightweight governance for adoption.
Consider following up with `/design-workflow` to adapt the playbook for a specific team.
